### 一、输入部分

#### 1. 构建独热向量

对于输入部分，首先在预训练阶段会构建一个词表，该词表用来对输入文本的 token 进行独热向量编码。

例如句子：**"I love you"**

由文本切分器进行 token 切分，切为 **"I"**、**"love"**、**"you"** 三个 token，对照词表构建一个 V 维的独热向量（V 一般为几万）。

#### 2. 构建词嵌入向量

每个 token 的独热向量只有一个维度有数值，其余地方都是 0，信息密度很低。

通过一个预训练好的词嵌入矩阵与独热向量相乘，可以：
- **降低向量维度**：从 V 维降为 d 维（通常 d=512）
- **获取语义信息**：512 维向量表示 512 个不同语义方面的信息
- **语义空间定位**：每个 token 在特定的语义坐标轴上有唯一位置

#### 3. 位置编码

Transformer 没有循环结构，需要显式地为每个 token 添加位置信息。

根据 token 在序列中的位置 `pos`、不同维度 `i`、向量总维度 `d` 和正余弦公式，计算位置编码向量：

![位置编码公式](https://github.com/user-attachments/assets/9ec0eb53-0a46-42fd-a2c9-a0bf17a420d2)

将位置编码向量和词嵌入向量**逐元素相加**，得到的矩阵既包含词序信息，又包含丰富的语义信息。

#### 4. 生成 Q、K、V 矩阵

将嵌入矩阵分别与三个权重矩阵相乘，得到 Query（查询）、Key（键）、Value（值）三个矩阵：

$$ Q = X \cdot W^Q $$
$$ K = X \cdot W^K $$
$$ V = X \cdot W^V $$

#### 5. 计算注意力权重矩阵

**第一步**：计算 Q 和 K 的点积，得到原始注意力分数矩阵

**第二步**：将注意力分数矩阵除以 $\sqrt{d_k}$，防止点积结果过大

> **作用**：
> 1. 使最大的值和第二大的值能拉开适度差距（不会太挤，也不会太极端）
> 2. 让数值落在 softmax 最灵敏的区域，梯度最大，学习最快

**第三步**：对注意力分数矩阵进行 softmax 操作，得到注意力权重矩阵

**第四步**：将注意力权重矩阵与 V 矩阵相乘，得到注意力输出矩阵

$$ Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

#### 6. 计算多头注意力

![多头注意力结构](image-1.png)

**步骤**：

1. **分头**：将 Q、K、V 矩阵均分为多个子矩阵（通常分为 8 个头）
2. **并行计算**：对每个子矩阵独立计算注意力权重
3. **拼接**：将多个头的输出结果进行 concat 拼接
4. **线性变换**：乘以输出权重矩阵 $W^O$，融合多头信息

> **作用**：让不同头学到的不同特征相互交互，增强模型的表达能力。

#### 7. 掩码自注意力（填充法）

在实际训练中，一个 batch 里的句子长度不一致，需要进行填充和掩码处理。

**第一步：填充**

使所有句子长度相同（为了方便批量计算）：

```
句子1: "我 爱 吃 苹 果"        → 长度 5
句子2: "你 好  [PAD] [PAD] [PAD]" → 长度 5（填充3个PAD）
句子3: "今 天 天 气 真 好"    → 长度 6
```

**第二步：生成掩码矩阵**

假设序列长度为 4，有效词位置为 1，填充位为 0：

|        | Key1 | Key2 | Key3 | Key4 |
| ------ | ---- | ---- | ---- | ---- |
| Query1 | 1    | 1    | 0    | 0    |
| Query2 | 1    | 1    | 0    | 0    |
| Query3 | 1    | 1    | 0    | 0    |
| Query4 | 1    | 1    | 0    | 0    |

> **注意**：Key3、Key4 是 PAD，所以所有 Query 都不能关注它们；Query3、Query4 本身是 PAD，它们的行也全掩掉。

**第三步：在 softmax 之前把掩码位置设为 -∞**

$$ M_{mask} = \begin{cases} 
0 & \text{有效位置（不影响 softmax）} \\
-\infty & \text{填充位置（softmax 后为 0）}
\end{cases} $$

**第四步：应用掩码后的 softmax**

|        | Key1 | Key2 | Key3 | Key4 |
| ------ | ---- | ---- | ---- | ---- |
| Query1 | 2.3  | 1.5  | -∞   | -∞   |
| Query2 | 0.8  | 2.1  | -∞   | -∞   |
| Query3 | -∞   | -∞   | -∞   | -∞   |
| Query4 | -∞   | -∞   | -∞   | -∞   |

↓ softmax 后 ↓

|        | Key1 | Key2 | Key3 | Key4 |
| ------ | ---- | ---- | ---- | ---- |
| Query1 | 0.69 | 0.31 | 0    | 0    |
| Query2 | 0.21 | 0.79 | 0    | 0    |
| Query3 | 0.25 | 0.25 | 0.25 | 0.25 |
| Query4 | 0.25 | 0.25 | 0.25 | 0.25 |

> **说明**：Query3、Query4 虽然 softmax 后有值，但后续计算会忽略这些位置的输出。

#### 8. 层归一化

将多头注意力输出矩阵进行层归一化（Layer Normalization），稳定训练过程：

$$ LN(x) = \alpha \cdot \frac{x - \mu}{\sigma + \epsilon} + \beta $$

其中 $\mu$ 是均值，$\sigma$ 是标准差，$\alpha$ 和 $\beta$ 是可学习的参数。

#### 9. 残差连接

将层归一化后的矩阵与输入矩阵进行残差连接（Residual Connection）：

$$ Output = LN(x + Attention(x)) $$

> **作用**：缓解梯度消失问题，使深层网络的训练更加稳定。

### 三、整体流程总结

```
输入文本
    ↓
Token 切分
    ↓
独热向量编码
    ↓
词嵌入（Embedding）
    ↓
位置编码（Positional Encoding）
    ↓
生成 Q、K、V 矩阵
    ↓
多头注意力机制
    ↓
层归一化 + 残差连接
    ↓
前馈神经网络（FFN）
    ↓
层归一化 + 残差连接
    ↓
输出层（线性 + softmax）
    ↓
输出结果
```
